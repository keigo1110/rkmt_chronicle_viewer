おすすめは Next.js（Vercel）＋ OpenSeadragon（Deep Zoom）＋ OCRインデックス（JSON）です。理由は、1枚作品の鑑賞体験（高速ズーム・パン、滑らか、ミニマップ、モバイルジェスチャ）が最短で“地図品質”に到達し、OCR検索→その位置へジャンプ（＋ハイライト）も実装が素直だからです。

全体方針（UX基準）
・初期表示は軽い（低解像タイルだけ先に出る）
・拡大してもシャープ（必要タイルだけ読み込み）
・検索は「即時候補→Enterでジャンプ→該当箇所を矩形で強調」
・URL共有できる（`?q=xxx` で同じ場所へ飛べる）
・モバイルで破綻しない（ピンチズーム、慣性、誤タップ対策）

1. アセット生成（ローカルで1回やってコミット）
   (1) タイル化（Deep Zoom）

* 元画像（10198×3164）→ DZI + タイル群を生成
* タイルサイズは 256 推奨（CDN/ブラウザキャッシュ効率が良い）

例（vips）：

```bash
vips dzsave オープンハウス年表sat.png public/tiles/timeline \
  --tile-size=256 --overlap=1 --suffix=.jpg[Q=85]
# public/tiles/timeline.dzi と public/tiles/timeline_files/ ができる
```

(2) OCR（検索用データを作る）
ここは「精度優先」ならクラウドOCRを1回だけ使ってJSON化してしまうのが一番ラクで堅いです（運用中に課金が走る形にはしない）。ローカル完結なら Tesseract + 前処理でいけますが、細かい文字は取りこぼしが出やすいので、作品体験の品質を取るならクラウド1回実行を推します。

出力はこういう形に統一します（重要なのは bbox が“元画像ピクセル座標”なこと）：

```json
[
  { "text": "Incremental Gaussian Splatting", "bbox": [8120, 1680, 520, 42], "conf": 0.93, "norm": "incremental gaussian splatting" },
  ...
]
```

2. Next.js（Vercel）側の実装（React）
   (1) ビューア

* OpenSeadragonで `public/tiles/timeline.dzi` を読む
* `showNavigator: true`（ミニマップ）をON
* 背景は黒、UIは邪魔しない最小限（作品優先）

(2) 検索UI

* 画面上部に検索ボックス（/キーでフォーカス、Escで閉じる）
* 入力と同時に候補リスト表示（10件程度）
* 曖昧検索は Fuse.js が速くて扱いやすい（`norm` を対象に）

(3) “その位置へ飛ぶ”

* 検索で選ばれたエントリの bbox を OpenSeadragon に渡して `fitBounds`（寄って見せる）
* 同時に bbox の矩形を overlay として描画して 1〜2秒だけ強調（体験が一気に地図っぽくなります）
* 同一クエリで複数ヒットしたら「次へ/前へ」ボタン or n/p キーで巡回

(4) 共有リンク

* `/?q=gaussian` でページを開いたら自動検索して最初のヒットへジャンプ
* さらにこだわるなら `/?x=...&y=...&z=...`（カメラ位置）も保存できますが、まずは `q` だけで十分UX良いです

3. デプロイ設計（Vercelで安定させる）
   基本は「全部 static」に寄せます。
   ・`public/tiles/**`：タイルと `.dzi` を静的配信
   ・`public/ocr/entries.json`：OCRインデックス（静的配信）
   ・Next.js は静的ページ＋クライアントでビューア起動（API不要）

注意点として、タイルのファイル数や合計サイズが増えた場合は、作品ページはVercelのまま、タイルだけ Cloudflare R2 / S3 + CDN に逃がすのが王道です（URLを差し替えるだけでUXはそのまま）。今回の解像度規模だと、まずはVercel直置きで試して十分通るケースが多いです。

4. 仕上げの“プロ品質”チェック項目（ここで差が出ます）
   ・タイルのキャッシュ：長めに効かせる（更新時はパスを変える運用がラク）
   ・初期ズーム位置：作品全体が気持ちよく収まるスケールで開始
   ・最大ズーム：過剰に拡大しすぎない上限（荒れ始める倍率で止める）
   ・モバイル：検索候補がズーム操作を邪魔しない、誤タップで候補が消えない
   ・OCR補正：作品タイトルや固有名詞だけ手動辞書（別JSON）で上書きできるようにしておくと、体験が一段上がります

5. 最終成果物（最小セット）
   ・`public/tiles/timeline.dzi` と `public/tiles/timeline_files/**`
   ・`public/ocr/entries.json`（bbox付き）
   ・Next.js ページ：Viewer＋Search＋Jump＋Highlight＋Share（`?q=`）

このプランで行けば、「地図っぽい区画読み込み」「OCR検索」「ヒット位置へジャンプ」「Vercelで公開」まで、全部一貫して強いUXにできます。次のアクションとしては、(A) タイル生成、(B) OCR JSON生成、(C) Next.jsビューア実装、の順で詰めるのが最短です。必要なら、この構成のフォルダ設計と scripts（tile/o cr生成コマンド、Next.jsのコンポーネント雛形）まで一気に固定案を出します。
